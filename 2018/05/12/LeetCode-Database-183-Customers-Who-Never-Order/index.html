[LeetCode/Database] 183 Customers Who Never Order

Posted on 2018-05-12 | In Coding

Write a SQL query to find all customers who never order anything.

Problem

Suppose that a website contains two tables, the Customers table and the Orders table. Write a SQL query to find all customers who never order anything.

Table: Customers

+----+-------+
| Id | Name  |
+----+-------+
| 1  | Joe   |
| 2  | Henry |
| 3  | Sam   |
| 4  | Max   |
+----+-------+
Table: Orders 
+----+------------+
| Id | CustomerId |
+----+------------+
| 1  | 3          |
| 2  | 1          |
+----+------------+
Using the above tables as example, return the following: 
+-----------+
| Customers |
+-----------+
| Henry     |
| Max       |
+-----------+

Solution

Solution 1: Using LEFT JOIN

SELECT Name AS Customers
FROM Customers LEFT JOIN Orders
ON Customers.Id = Orders.CustomerId
WHERE Orders.CustomerId is NULL;

Solution 2: Using sub-query and NOT IN

SELECT Name AS Customers FROM Customers
WHERE Customers.Id NOT IN (SELECT CustomerId FROM Orders);

Related Knowledge

LEFT JOIN

The result of a left outer join (or simply left join) for tables A and B always contains all rows of the “left” table (A), even if the join-condition does not find any matching row in the “right” table (B).

See Related Knowledge in 175 Combine Two Tables for more information on JOIN clauses.

IN & NOT IN

The MySQL [NOT] IN condition is used to help reduce the need to use multiple OR conditions.
The syntax for the [NOT] IN condition in MySQL is:
expression [NOT] IN (value1, value2, …. value_n);
Note that we can also use a sub-query after the keyword [NOT]IN, just like what we did in Solution 2.

Conclusion

The intuition for Solution 1 is to think of the vein diagram for this problem.
For Solution 2, we frist find out customers who have ordered. Then, we can use NOT IN to query the customers who are not in this list.
