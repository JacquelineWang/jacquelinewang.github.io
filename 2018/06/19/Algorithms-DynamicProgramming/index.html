[Algorithms] Dynamic Programming

Posted on 2018-06-19 | In Coding

Dynamic Programming is an algorithmic paradigm that solves a given complex problem by breaking it into subproblems and stores the results of subproblems to avoid computing the same results again.

Concepts

Following are the two main properties of a problem that suggests that the given problem can be solved using Dynamic programming.

Overlapping

Dynamic Programming is mainly used when solutions of same subproblems are needed again and again. In dynamic programming, computed solutions to subproblems are stored in a table so that these don’t have to be recomputed.

Optimal Substructure

A given problems has Optimal Substructure Property if the optimal solution is the combination of its subproblems’ optimals.

For example, the Shortest Path problem has following optimal substructure property:
If a node x lies in the shortest path from node u to node v, then minPath(u,v) = minPath(u,x) + minPath(u,v).

On the other hand, the Longest Path problem doesn’t have the Optimal Substructure property. Here by Longest Path we mean longest simple path (path without cycle) between two nodes. For example, there are two longest paths from q to t: q→r→t and q→s→t. However, maxPath(q,t) != maxPath(q,r) + maxPath(r,t), because the longest path from q to r is q→s→t→r and the longest path from r to t is r→q→s→t.

Solutions

There are following two different ways to store the values so that these values can be reused: Memoization and Tabulation.
The time taken by these two DP solutions is much less than the exponential brute Recursive method.
Note that unlike the Tabulated version, all entries of the lookup table are not necessarily filled in Memoized version.

Solution 1: Memorization

  • Top down
  • Look-up table

Solution 2: Tabulation

  • Bottom up
  • First calculate f(0), then f(1), then f(2)…

Steps

  1. Think backward, calculate forward
  2. Write down the recursive formula: currentState = lastState + currentMove
  3. Deal with corner case: lastState have various posibilities. We should distinguish the firstState(which could also have multiple choices) to begin with
  4. Add hash table: record every currentState

Best Time to Buy and Sell Stock (121)

Problem:
Say you have an array for which the ith element is the price of a given stock on day i.
If you were only permitted to complete at most one transaction (i.e., buy one and sell one share of the stock), design an algorithm to find the maximum profit.
Note that you cannot sell a stock before you buy one.

Example:

Input: [7,1,5,3,6,4]
Output: 5
Explanation: Buy on day 2 (price = 1) and sell on day 5 (price = 6), profit = 6-1 = 5.
Not 7-1 = 6, as selling price needs to be larger than buying price.
